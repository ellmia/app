# エルミア 仕様書

> **作成背景**: Matt Pocock の "grill-me" メソッド（深掘り質問による要件明確化）により複数ラウンドで採択された仕様を記録したもの。
> このドキュメントが本プロジェクトの **Single Source of Truth** となる。

## 1. 製品概要

**製品名**: エルミア

**コンセプト**:
イケメンホストがソープ嬢（主にソープランド従事者）に対して、実践的で現実的なコンサルティングを行うチャット Web UI。

**ターゲットユーザー**:
- ソープランドで働く女性（新人〜ベテラン）
- 集客・単価・安全・メンタル・キャリア判断などで悩んでいる人

**価値提案**:
- ただの優しい先輩ではなく、「夜の仕事のリアルを知った男性視点 + 女性の味方」という独特の立場からアドバイスを提供
- 無料枠の OpenRouter を利用し、低コストで高品質な相談体験を実現

## 2. Grill-me メソッドで採択された主要決定事項

### 2.1 ブランディング・名称
- **採択**: 「エルミア」
  - 副題例: 「イケメンホストのソープコンサル」
- URL: `https://app.lmia.workers.dev`
- リポジトリ: `https://github.com/ellmia/app`

### 2.2 ペルソナ（最も重要）
- **採択**: 「イケメンホスト」（Gentlemanly cool host）
  - 話し方: タメ口ベースだが、敬意とケアを込めたもの。軽くチャーミング。
  - 態度: 押し付けがましくなく、対話的。「一緒に考えよう」というスタンス。
  - 絶対遵守原則（システムプロンプトに明記）:
    1. 安全・健康・メンタル最優先
    2. 違法・非同意・危険行為は即座に強く拒否
    3. 現実的で具体的なアクションを提示（今日からできること）
    4. 相手の状況（店所属/独立、経験年数など）を尊重
    5. メンタルケアを自然に織り交ぜる
    6. 業界のリアル（客単価、 burnout、クレーム対応など）を正しく扱う

### 2.3 スコープ（MVP として採択）
- **Ultra Minimal**（ユーザーが明確に選択）
  - フルスクリーンのチャット UI のみ（携帯特化、ライトモード）
  - ヘッダー: ロゴ（LMIA） + 「新しい相談」ボタン
  - UI ブランド: ユニコーン世界観（🦄、モデルごとのキャラ名表示: ジェミー・久遠 等）
  - ハードコードされた高品質なシステムプロンプト（イケメンホスト）
  - OpenRouter によるストリーミング応答
  - **1問1答**（履歴を蓄積せず単発相談で精度を優先）
  - 回答再生成ボタン（「ねぇ、酔いすぎ！ちゃんと答えて！」）
  - アシスタント応答のマークダウン表示（`react-markdown` + `remark-gfm`）
  - 例の質問チップ 30 件（初期表示）
  - Google Analytics（`Layout.astro`、測定 ID: `G-40LT2E7S01`）
  - 免責事項の明記（UI とプロンプトの両方）

**非採用（MVP では入れない）**:
- 事前プロフィール入力フォーム（経験年数・目標など）
- チャット履歴の永続化（localStorage すら最小限）
- ユーザーアカウント・ログイン
- 複数ペルソナ切り替え（UI 上の手動切替）
- 画像生成

### 2.4 LLM / OpenRouter 関連
- **APIキー運用**: 開発者（ユーザー）が取得した OpenRouter の無料枠/クレジットを使用
  - サーバーサイドで保持（クライアントには渡さない）
  - 無料枠保護のため **IPベースのレートリミット** を必須で実装
- **モデル設定**（grill-me で採択）: 
  - ローカル `.env` と本番で `LLM_MODEL=openrouter/free` を統一して使用（OpenRouter の無料モデル自動選択）
  - デフォルト（未設定時）: `openrouter/free`（`astro.config.mjs` の env schema でも同値）
  - 本番設定方法: `npx wrangler secret put LLM_MODEL`（値に `openrouter/free` を入力）
  - ローカル: `.env` に `LLM_MODEL=openrouter/free` と記載
- ストリーミング: 必須（レスポンシブなチャット体験のため）

### 2.5 技術スタック・開発規約
- **完全準拠**:
  - Astro 6（`output: "server"`）
  - Cloudflare Workers（`wrangler deploy`）
  - React Islands（インタラクティブ部分のみ `.tsx`）
  - Tailwind CSS 4（`@tailwindcss/vite`）
  - 厳格 TypeScript（`any` 禁止、`@/*` エイリアス）
- **開発フロー**:
  - 変更後は必ず `npm run check`（`astro check`）
  - ビルドは `astro check && astro build`
  - 既存プロジェクト（特に `ai/` と `paperlevels/`）の構成・規約を最大限再利用
- **エラーハンドリング**:
  - すべての API レスポンスは `{ error: string }` 形式
  - 外部 API 呼び出し（特に OpenRouter）には必ず `AbortController` でタイムアウト設定

### 2.6 セキュリティ・運用
- レートリミット: メモリベース（isolate 単位）で十分な保護（後で KV 強化可）
- シークレット管理:
  - 開発: `.env` または `.dev.vars`
  - 本番: 
    - `npx wrangler secret put OPENROUTER_API_KEY`
    - `LLM_MODEL=openrouter/free` を本番でも使用する場合: `npx wrangler secret put LLM_MODEL`（値に `openrouter/free` を入力）
- 免責: UI とシステムプロンプトの両方で明確に記載

## 4. 非機能要件（採択）

- パフォーマンス: ストリーミングで体感速度を優先
- コスト: ユーザーの無料枠を最優先に守る（レートリミット + 安価モデル）
- 保守性: 既存の `ai` / `paperlevels` エコシステムに溶け込む
- 倫理: 安全・同意・違法行為の拒否を最優先

## 5. 将来の拡張可能性（MVP ではスコープ外だが記録）

- チャット履歴の localStorage 保存 / エクスポート
- ユーザー状況の簡易プロフィール注入（システムプロンプトへの動的追加）
- 複数のコンサルタントペルソナ（女性先輩バージョンなど）
- より高度なレートリミット（KV + 1日上限）
